# Code of Conduct

## Our standard

Atlas is built on the premise that clear, honest reasoning beats confident-sounding certainty. We hold ourselves to the same standard internally.

- Disagree on substance, not personality. If you think an architectural or product decision is wrong, say why, with evidence — the same bar we hold the Confidence Framework to.
- No harassment, discrimination, or personal attacks, in issues, PRs, reviews, or any project space.
- Assume good faith, but don't let "good faith" become a reason to avoid direct feedback.

## Enforcement

Violations can be reported to the maintainers privately (see `SECURITY.md` for the contact pattern — replace with a real conduct-specific address before launch). Maintainers will respond proportionally, from a private warning up to removal from the project.

Adapted in spirit from the Contributor Covenant.
